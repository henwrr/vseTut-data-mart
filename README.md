# Витрина данных маркетплейса «ВсёТут»

## Описание проекта

Разработка витрины данных маркетплейса на основе заказов пользователей и решение ad hoc аналитических задач с помощью SQL.

**Инструмент:** PostgreSQL / DBeaver
**Схема данных:** `ds_ecom`

---

## Витрина данных `product_user_features`

### Условия формирования
- Только заказы со статусом **«Доставлено»** или **«Отменено»**
- Только **три региона** с наибольшим количеством заказов
- Каждая строка — уникальная пара **«клиент — регион»**

### Структура витрины

| Поле | Описание |
|------|----------|
| `user_id` | Уникальный идентификатор клиента |
| `region` | Регион заказа |
| `first_order_ts` | Дата и время первого заказа |
| `last_order_ts` | Дата и время последнего заказа |
| `lifetime` | Жизненный цикл клиента (дней между первым и последним заказом) |
| `total_orders` | Общее количество заказов |
| `avg_order_rating` | Средняя оценка заказов |
| `num_orders_with_rating` | Количество заказов с оценкой |
| `num_canceled_orders` | Количество отменённых заказов |
| `canceled_orders_ratio` | Доля отменённых заказов |
| `total_order_costs` | Суммарная стоимость доставленных заказов |
| `avg_order_cost` | Средняя стоимость заказа |
| `num_installment_orders` | Количество заказов в рассрочку |
| `num_orders_with_promo` | Количество заказов с промокодом |
| `used_money_transfer` | Использовал ли клиент денежный перевод (1/0) |
| `used_installments` | Использовал ли клиент рассрочку (1/0) |
| `used_cancel` | Отменял ли клиент заказы (1/0) |

## Ad Hoc задачи

### Задача 1. Сегментация пользователей по количеству заказов

| Сегмент | Пользователей | Среднее заказов | Средний чек |
|---------|--------------|-----------------|-------------|
| 1 заказ | 60 460 | 1.00 | 3 324 ₽ |
| 2–5 заказов | 1 934 | 2.09 | 3 091 ₽ |
| 6–10 заказов | 5 | 7.00 | 2 773 ₽ |
| 11+ заказов | 1 | 15.00 | 1 245 ₽ |

**Вывод:** Подавляющее большинство клиентов (97%) совершают только 1 заказ. Средний чек снижается с ростом числа заказов — лояльные клиенты покупают чаще, но дешевле.

---

### Задача 2. Топ-15 пользователей по среднему чеку (от 3 заказов)

Средний чек топ-15 пользователей: от **5 527 ₽** до **14 717 ₽**.

| Регион | Кол-во пользователей |
|--------|---------------------|
| Москва | 9 |
| Санкт-Петербург | 4 |
| Новосибирская область | 2 |

**Вывод:** Москва доминирует среди высокочековых клиентов. Большинство из топ-15 сделали ровно 3 заказа — высокий чек не всегда связан с высокой частотой покупок.

---

### Задача 3. Статистика по регионам

| Регион | Клиентов | Заказов | Средний чек | Рассрочка | Промокоды | Отмены |
|--------|----------|---------|-------------|-----------|-----------|--------|
| Москва | 39 386 | 40 747 | 3 167 ₽ | 47.73% | 3.96% | 0.51% |
| Санкт-Петербург | — | — | 3 620 ₽ | 54.66% | 4.16% | — |
| Новосибирская область | — | — | 3 519 ₽ | 54.14% | 3.68% | 0.43% |

**Вывод:** Москва лидирует по объёму, но Санкт-Петербург — по качеству: самый высокий средний чек, наибольшая доля рассрочки и промокодов. Новосибирская область — наименьший процент отмен.

---

### Задача 4. Активность пользователей по месяцу первого заказа в 2023 году

**Ключевые наблюдения:**
- **Пик притока:** ноябрь (4 703 пользователя) — сезонность, декабрь (3 589)
- **Самый высокий средний чек:** сентябрь (3 312 ₽), самый низкий — февраль (2 581 ₽)
- **Рейтинг:** стабильно высокий (4.0–4.3), наивысший в августе (4.32)
- **Денежные переводы:** 19–22% пользователей, пик в феврале (22.11%)
- **Активность:** январские пользователи активны 12.8 дней, ноябрьские — около 2 дней

**Вывод:** Осенние пользователи (сен–окт) — наиболее платёжеспособные. Ноябрьский всплеск привлекает много пользователей, но с низкой лояльностью. Пользователи начала года — самые лояльные.
